### 为什么要做 Code Review

![](http://i.imgur.com/gGwno33.gif)

- 给 Coder 足够压力(工程师最怕自己代码给别人看)

- 给 Reviewer 偷懒时间(工程师最喜欢指点别人代码)

- 不给老板扣工资机会(找到逻辑错误)

- 组建一家技术牛逼的团队(保持团队技术讨论氛围)

### Code Review 的建议

- Merge 之前做 Review

- 单元测试通过之后做 Review

- Review 功能要写明，代码要可读

- 先找逻辑问题，再找风格问题

- 避免过多争论主观意向很强的内容
